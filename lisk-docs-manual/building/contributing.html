Contributing to the documentation

Creating issues

Working on an issue

  1. To work on an issue, first assign yourself to it.

  2. Then check out docs-sdk and create a new branch named after the issue you are working on:

    git checkout docs-sdk
git checkout -b 123-issue-title

    where 123 should be the corresponding issue number.

  3. Now commit the intended changes and push them back to the lisk-docs repository on Github:

    git push origin 123-issue-title

Creating a PR (pull request)

  1. On Github, now open a PR with docs-sdk as base branch, add the PR to the project created in section Preparing the release. Then link the corresponding issue in the description and request a reviewer.
